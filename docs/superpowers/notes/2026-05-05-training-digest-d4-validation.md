## Method

report; cross-referenced against `data/digests/training/*.json` and
`phase3_factorial_summary.json`. Each claim classified as EXACT, CLOSE, or
MISSING. MISSING claims further classified as parser-fixable (v1.1),
schema-extension (v2), or out-of-scope (mechanism / stats / bench).

All numeric comparisons done by direct calculation from digest JSON fields.
Rounding tolerance: 4+ significant figures treated as EXACT.

## Gaps and follow-ups

### Parser-fixable (v1.1)

None. The `entropy_penalty_trace` gap is a **deliberate v1 stub** — the parser code comments "Task 9 stub — entropy-penalty parsing deferred to v2" and the spec (`docs/superpowers/specs/2026-05-05-training-digest-observer.md`) explicitly includes the `entropy_penalty_trace` schema fields (`tau_anneal_observed_step_range`, `beta_H_eff_observed_range`, `H_observed_range`) that were designed but not implemented in v1. This is correctly classified as a schema-extension gap, not a parser bug.

### Schema-extensions (v2)

1. **TODO-D4-1: entropy_penalty_trace population for MoE cells.**
   Field: `entropy_penalty_trace.{applicable, H_observed_range, tau_anneal_observed_step_range, beta_H_eff_observed_range, entropy_penalty_loss_sum}`.
   The spec already defines the schema. Implementation requires parsing `[entpen] step=N ...` lines from MoE cell stdouts and computing mean/min/max H, tau, beta_H_eff per cell. The mechanism report relies on per-cell H values (specifically the last-500-step window mean) to explain router behavior (§2.1, §3.2, §3.3). Without these values in the digest, the next P10 mechanism dispatch cannot derive router sharpness from digests alone.
   Affected cells: C42, C43, C44, D42, D43, D44.

2. **TODO-D4-2: Sub-window per-step lm_loss trajectory statistics.**
   The mechanism report's §2.2 trajectory decomposition uses 8 custom windows with arbitrary step boundaries ([0,200), [200,500), [500,1000), etc.). These cannot be derived from the 9 step-summary checkpoints at 500-step intervals.
   To support future mechanism extraction, v2 could add a `trajectory.window_stats` array capturing mean lm_loss for a fixed set of standard analysis windows. Alternatively, this is intentionally deferred — the mechanism extractor that needs sub-window analysis reads raw stdout for those specific cells, using the digest only for the well-defined endpoint metrics.
   Recommendation: defer to v2 with a note that per-step window analysis requires raw stdout; the 500-step summaries cover the key endpoints.

3. **TODO-D4-3: Per-step PPL distribution statistics within a terminal window.**
   The mechanism report §4.1 shows min/median/max single-batch PPL for specific cells in [4000,5000) (C42 min=3.49, median=356.48, max=6209.87 etc.). These are per-step single-batch PPL distributions unavailable from step summaries.
   This is low priority: the mechanism report correctly concludes the best_ppl (global run min) is the only field relevant for P10 disposition, and that field is EXACT in the digest. The within-window distribution stats are supporting evidence only.

### Out of scope (correct exclusions)

The following are confirmed correctly NOT captured by the digest — they are mechanism explanations, statistical tests, or P11-forward items:

- **D-vs-B mechanism explanation (§2.3):** per-expert data budget starvation derivation (4× fewer tokens/expert), k/K=25% calculation, "structural representational gap" narrative. These are P10's value-add.
- **B6 fixed-effects regression coefficients (§5.2):** LR_high β=−0.363, τ_anneal β=−2.868, β_H_eff β=−12.879, etc. Statistical analysis outputs; not training-time signals.
- **Holm step-down correction results (§5.3):** p-values, Holm-adjusted p-values, REJECT_H0/FAIL_TO_REJECT decisions. Statistical inference; digest scope is raw signals only.
- **Variance-component decomposition (§5.4):** σ²_seed=0.000367, σ²_LR-cell=0.000000. Mixed-effects analysis; out of scope.
- **FLAME-MoE reconciliation status (§5.6):** "NULL at P10 training-time signals". This is a P10 interpretation call.
- **Architecture × LR interaction (F=107, p=6.5e-6):** ANOVA from P9; digest does not replicate P9 statistical outputs by design (spec §"Explicitly NOT in aggregate").
- **Per-source LM-loss (§6):** Not emitted in stdout. The mechanism report §6.1 confirms 0 per-source loss markers; digest correctly captures only the byte-budget stream diagnostics which ARE in stdout.
- **C42 outlier sensitivity analysis (D-312):** PI's C42-out sensitivity check (Δ_arch 0.388→0.382, F_arch 3324→12210). Post-hoc statistical analysis; out of scope.
- **Router-basin lottery explanation (§3.3):** First-principles derivation of why LR_low fails to break symmetry. Mechanism narrative; not a signal.
- **Pre-reg §7.4 4-component router-health vector degeneracy finding (§3.5):** load-CV, kill_hit-rate, EMA-uniformity, τ-floor-time-fraction all pinned uniform. This IS partially captured: the digest records gate_hit_total=0, kill_hit_total=0, regime_class=uniform for all MoE cells (EXACT matches). But the interpretation that this makes the pre-reg vector "degenerate" is mechanism_extractor's analytical conclusion, not a digest field.

## Conclusion

The digest pipeline is **ready for the next P10 mechanism dispatch** to consume instead of raw stdouts for all quantitative endpoint claims. The 15 EXACT + 2 CLOSE matches cover every cell-level endpoint metric the mechanism report uses as primary evidence: step-4500 avg_lm, best_ppl, seed-sd, MoE balance health (gate_hit/kill_hit/regime_class/imbalance_ranges), NaN classification, wall time, ETA overrun, cumulative wall, and stream byte budgets.

The 5 MISSING items fall into two non-blocking categories:

- **TODO-D4-1 (entropy_penalty_trace, v2)** is the most consequential gap. The per-cell routing entropy H values (H≈1.55–1.66 for MoE cells) are the mechanism report's key discriminating signal for router sharpness analysis (§2.1, §3.2, §3.3). A future P10 mechanism dispatch reading only digests would need to fall back to raw stdout to reproduce the H values in those three sections. The spec anticipated this (schema fields defined but deferred); implementation in v2 is the priority fix.

- **TODO-D4-2 and TODO-D4-3** (sub-window trajectory stats, per-step PPL distributions) are supporting-evidence gaps only. The mechanism report uses these to corroborate claims already established by the endpoint metrics (which the digest captures exactly). A mechanism dispatch can reproduce the primary claims from digests; the sub-window tables would require raw-stdout fallback for the trajectory decomposition sections only.

**Verdict: ready with documented gaps.** The digest pipeline eliminates the need to read raw stdouts for ~80% of mechanism report content. The 20% gap (entropy H values + sub-window trajectories) is documented as TODO-D4-1/2/3 for v2 spec work. The next P10 dispatch should read digests first per the mechanism_extractor procedural (wired in D3), then fall back to raw stdout only for entropy H and sub-window trajectory sections.
